# Contributing

## Branch Strategy

- main
- develop
- feature/*

## Commit Messages

Use descriptive commit messages.

Examples:

- feat(common): add baseline OS role
- feat(postgresql): automate PostgreSQL installation
- docs: update deployment guide
- fix(web): correct nginx configuration

## Code Standards

- Use Ansible Core 2.14 syntax.
- Use Fully Qualified Collection Names (FQCN).
- Keep tasks idempotent.
- Test changes before committing.
